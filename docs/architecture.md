# Архитектура backend-сервиса

## Цель системы

Сервис обеспечивает переход от прогноза объёма отгрузок к конкретному операционному действию: расчёту количества транспорта и формированию заявки на вызов машин.

## Компоненты

### 1. Go API

Роль:

- orchestration;
- decision logic;
- batch pipeline;
- расчёт метрик;
- демонстрационный UI;
- сбор и проксирование телеметрии.

### 2. Python ML service

Роль:

- inference по временным точкам;
- выбор модели;
- адаптация входных конкурсных датасетов;
- fallback-режим при отсутствии модели.

### 3. UI

Роль:

- показать end-to-end контур на защите;
- быстро проверить API вручную;
- показать живые логи Go и Python;
- показать response payload'ы без дополнительных инструментов.

## Потоки

### Онлайн-контур

```text
client -> /predict -> ML service -> /decision -> truck request
```

### Альтернативный онлайн-контур

```text
client -> /decision(points) -> Go API -> ML service -> decision engine
```

### Офлайн-контур

```text
test_team_track.parquet
  -> /batch
  -> ML dataset adapter
  -> /predict
  -> submission.csv
```

### Контур оценки

```text
predictions + actuals -> /metrics -> WAPE + Relative Bias
```

### Контур демонстрации

```text
browser -> Go UI -> Go API -> Python ML service
```

## Бизнес-логика продукта

- шаг планирования: 30 минут;
- горизонт решения: 2 часа;
- основной сигнал: прогноз `target_2h`;
- корректировка: safety factor;
- ограничение: `max_trucks_per_route`;
- базовое допущение: парк однороден по вместимости.

## Почему такая декомпозиция

Разделение на Go и Python позволяет:

- независимо масштабировать inference и orchestration;
- не смешивать ML-зависимости с API-слоем;
- упрощать замену модели без переписывания decision layer;
- оставлять бизнес-логику транспорта внутри основного сервиса.

## Масштабирование

- горизонтальное масштабирование Go API за балансировщиком;
- отдельное горизонтальное масштабирование ML service;
- вынос feature preparation в отдельный контур;
- подключение кэша горячих прогнозов;
- перевод на очередь событий для массовых batch-процессов.
